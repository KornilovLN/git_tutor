# Как создать и работать в новой ветке на Git

;---------------------------------------------------------------------

## Создайте новую ветку в локальном репозитории
## на основе текущей ветки (предположим, что это main):
```
git checkout -b new-feature
```

### Это создаст новую ветку new-feature и переключит вас на нее.

## Внесите необходимые изменения в ваш код,
## добавляйте новые файлы и коммитьте их в эту ветку new-feature:
```
git add .
git commit -m "Add new feature"
```


## Отправьте (push) new-feature в удаленный репозиторий на GitHub:
```
git push -u origin new-feature
```


### Флаг -u устанавливает upstream (отслеживаемую ветку) для новой
### ветки new-feature в удаленном репозитории.
### Это позволит вам в дальнейшем использовать более короткую 
### команду git push для отправки изменений в эту ветку.

### После выполнения этой команды новая ветка new-feature будет создана
### в вашем удаленном репозитории на GitHub,
### и все ваши локальные коммиты из этой ветки будут отправлены туда.

## Продолжайте работать в ветке new-feature, добавляя новые коммиты.
## Для отправки новых изменений в удаленный репозиторий, используйте:
```
git push
```

### Эта команда отправит все новые коммиты из вашей локальной ветки
### new-feature в соответствующую удаленную ветку new-feature на GitHub.


## Когда вы завершите работу над новой функциональностью и захотите
## объединить ее с основной веткой (main),
## перейдите в основную ветку локально:
```
git checkout main
```


## И выполните слияние (merge) ветки new-feature в main:
```
git merge new-feature
```


## Затем отправьте изменения в основную ветку на GitHub:
```
git push
```


### Таким образом, вы сможете работать над новой функциональностью в изолированной ветке new-feature,
### сохраняя ее на GitHub, и когда она будет готова, объединить ее с основной веткой main как локально,
### так и на удаленном репозитории GitHub.

### Не забывайте регулярно отправлять ваши коммиты в удаленный репозиторий с помощью git push, чтобы
### иметь резервную копию вашей работы на GitHub и возможность совместной работы с другими
### разработчиками, если это необходимо.



## Проверка синхронизации и отправка на github

```
Команда git branch:

    Команда git branch выводит список всех локальных веток и отмечает текущую ветку звездочкой *.

git branch

  master
* feature-branch
  another-branch


Команда git status:

    Команда git status выводит информацию о текущем состоянии репозитория, включая имя текущей ветки.
    
git status

On branch feature-branch
nothing to commit, working tree clean


Проверка веток на удаленном репозитории:

    Посмотрите, какие ветки существуют на удаленном репозитории:
    
git branch -r


Отправка текущей ветки на удаленный репозиторий:

    Если вы хотите, чтобы ветка new-feature появилась на GitHub, отправьте ее:
    
git push origin new-feature


Проверка ветки по умолчанию на GitHub:

    На GitHub может быть установлена другая ветка по умолчанию, например, develop.
    Вы можете изменить ветку по умолчанию в настройках репозитория:
        Перейдите в репозиторий на GitHub.
        Нажмите на вкладку "Settings" (Настройки).
        В разделе "Branches" (Ветки) вы можете увидеть текущую ветку по умолчанию
        и изменить её при необходимости.


Проверка синхронизации веток:

    Убедитесь, что локальная и удаленная ветки синхронизированы.
    Вы можете использовать команду git fetch для обновления информации о ветках:
        
git fetch



Полный процесс:

    Убедитесь, что вы находитесь в нужной ветке:

    bash

git checkout new-feature

    Отправьте изменения на удаленный репозиторий:

git push origin new-feature

    Установите отслеживание для новой ветки (если необходимо):

git push -u origin new-feature

    Проверьте ветки на удаленном репозитории:

git branch -r

Следуя этим шагам, вы сможете убедиться, что ваша локальная ветка
и ветка на GitHub синхронизированы должным образом.
    
    
	

    Проверка текущей ветки локально:

git branch

Вывод указывает, что вы находитесь в ветке new-feature, что верно.

    Проверка текущей отслеживаемой ветки:
    Убедитесь, что локальная ветка new-feature отслеживает правильную удаленную ветку.

git branch -vv

    Вы должны увидеть что-то вроде:    
    
* new-feature 1234567 [origin/new-feature] Your commit message

Убедитесь, что ветка develop действительно существует на GitHub:
Если на GitHub показывается ветка develop, проверьте это:

    Перейдите на страницу вашего репозитория на GitHub.
    Перейдите в раздел "Branches" (Ветки) и убедитесь, что ветка develop существует.

Смена ветки по умолчанию на GitHub (если необходимо):

    Если вам нужно изменить ветку по умолчанию с develop на new-feature или main, сделайте это в настройках репозитория на GitHub.
        Перейдите в раздел "Settings" (Настройки) вашего репозитория.
        Нажмите на "Branches" (Ветки) в меню слева.
        В разделе "Default branch" (Ветка по умолчанию) выберите нужную ветку и сохраните изменения.

Синхронизация изменений:

    Убедитесь, что все ваши изменения синхронизированы:

git fetch origin
git pull origin new-feature

Push в ветку new-feature:

    Если вы сделали изменения в ветке new-feature, отправьте их на GitHub:

git push origin new-feature
    
```
